# 进程与线程
## 进程
* 进程是程序在计算机中的一次执行实例。
* 系统分配资源（CPU、内存、I/O）给进程，是操作系统分配资源的基本单位。

## 线程
* 一个进程可以包含多个线程，一个线程也可以理解为一个进程的子进程。
* 一个线程就是一个指令流，将指令流中的一条条指令以一定的顺序交给CPU执行
* Java中，线程是最小调度单位

## 对比

| 特性   | 进程  | 线程  |
| -- | -- | --|
| 资源 | 拥有独立的内存空间和系统资源  | 与同属进程的线程共享内存空间 |
| 调度单位 | 是操作系统调度的基本单位  | 是程序内部调度的基本单位     |
| 通信         | 进程间通信开销较大：需要通过OS提供的 IPC（Inter-Process Communication）或者网络通信（如 HTTP，不同计算机之间通信）来交换信息 | 相对简单：同一进程内的线程共享内存空间，线程之间可以直接读写共享数据 |
| 开销 | 创建和切换开销较大 | 创建和切换开销较小  |
| 独立性 | 相互独立，互不影响 | 相互依赖，影响同一进程内其他线程 |
| 适用场景 | 程序并发执行、隔离任务   | 任务内部并行、提高 CPU 利用率 |
