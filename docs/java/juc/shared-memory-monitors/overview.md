# 共享模型之管程

## 本章概述

**管程（Monitor）** 是 Java 并发编程中最基础也最重要的同步机制。本章深入讲解如何使用管程解决多线程共享内存时的线程安全问题。

## 学习目标

通过本章学习，你将掌握：

- 理解多线程共享内存带来的**可见性、原子性、有序性**问题
- 掌握 `synchronized` 的使用与底层原理
- 理解 **Monitor（管程）** 的工作机制
- 掌握线程协作机制：`wait/notify` 和 `park/unpark`
- 学会分析和解决**死锁、活锁、饥饿**等活跃性问题
- 理解 `ReentrantLock` 显式锁的使用与优势
- 掌握常见的**并发设计模式**

## 内容结构

### 基础概念
- **共享带来的问题**：理解竞态条件、临界区等核心概念
- **线程安全分析**：学习如何分析线程安全问题
- **Monitor 原理**：深入理解管程的实现机制

### 同步工具

**synchronized 关键字**
- 基本用法与锁对象选择
- 字节码层面的实现原理
- 锁优化：偏向锁、轻量级锁、重量级锁

**ReentrantLock 显式锁**
- 可重入、可中断、超时获取锁
- 公平锁与非公平锁
- 与 synchronized 的对比

### 线程协作

**wait / notify 机制**
- 基于 Monitor 的等待通知机制
- 保护性暂停（Guarded Suspension）模式
- 生产者-消费者模式

**park / unpark 机制**
- LockSupport 的底层支持
- 与 wait/notify 的区别
- 应用场景

### 高级主题

**线程状态转换**
- 理解各种锁操作下的线程状态变化
- 深入分析状态转换的本质

**多把锁优化**
- 使用细粒度锁提升并发性能
- 避免锁竞争的设计技巧

**活跃性问题**
- 死锁的产生与预防
- 活锁与饥饿问题
- 哲学家就餐问题

**同步模式**
- 保护性暂停（Guarded Suspension）
- 固定运行顺序
- 交替输出控制

**异步模式**
- 生产者-消费者模式

## 学习建议

1. **循序渐进**：先理解概念，再学习工具，最后掌握模式
2. **动手实践**：每个知识点都配有代码示例，建议亲自运行调试
3. **关注原理**：不仅要知道怎么用，更要理解为什么这样设计
4. **对比分析**：理解不同工具的适用场景和优缺点
