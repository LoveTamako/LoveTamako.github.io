# 共享模型之内存

上一章主要关注的是访问共享变量时，保证临界区代码的**原子性**。

本章将深入学习共享变量在多线程间的**可见性**问题和多条指令执行时的**有序性**问题，从内存层面理解并发编程的本质。

- **可见性**：由 JVM 缓存优化引起
- **有序性**：由 JVM 指令重排序优化引起

## 学习目标

通过本章学习，你将掌握：

- 理解 **Java 内存模型（JMM）** 的工作原理
- 掌握多线程环境下的**可见性**和**有序性**问题及解决方案
- 深入理解 `volatile` 关键字的底层实现与应用场景
- 掌握 **happens-before 规则**在并发编程中的作用
- 学会使用 volatile 实现线程间的安全通信

## 主要内容

### Java 内存模型
- 主存与工作内存的抽象模型
- CPU 缓存、寄存器与内存的映射关系

### 可见性
- CPU 缓存导致的可见性问题
- synchronized 与 volatile 的可见性保证
- 两阶段终止模式的 volatile 实现

### 有序性
- 编译器优化与 CPU 指令重排序
- 单线程与多线程下的重排序影响
- 内存屏障的作用机制

### volatile 原理
- 写屏障与读屏障的工作原理
- 三大特性：可见性 ✅、有序性 ✅、原子性 ❌
- 双重检查锁定（DCL）为何必须使用 volatile
- **happens-before 规则**：Java 内存模型的核心可见性保证机制

::: tip 重点提示
volatile 是轻量级的同步机制。掌握本章内容后，你将能够：
- 正确使用 volatile 解决可见性和有序性问题
- 理解为什么某些看似安全的代码存在并发 bug
- 在合适的场景选择 volatile 而非重量级锁
:::
